## Multi-caller VCF requirement (cancer showcase)

> **TL;DR:** The cancer showcase MUST be regenerated from a combined VCF that
> contains SNV + CNV + SV callers. The current `sample_codegen_777_*` artifacts
> are **SNV-only** and will not surface CNV/SV findings such as CDKN2A/B loss.

### Why this matters

A FoundationOne CDx comparison (`_workspace/board-polish/foundation_one_comparison.md`)
flagged that the current `sample_codegen_777` showcase **misses CDKN2A/B biallelic
loss** — a finding present in 28–36% of pancreatic ductal adenocarcinomas. The
classification pipeline is not at fault: the input VCF was MuTect SNV-only, so
the CNV/SV channel had nothing to ingest. This is a **data-prep gap, not a
pipeline gap** — Canvas+Manta+AnnotSV+ACMG-CNV are already wired in
`scripts/orchestrate.py` and exercised by the `--sv` flag.

### Required inputs for a faithful cancer showcase

The real BIKO cancer pipeline expects a *multi-caller* picture of the tumor:

1. **SNV/indel VCF** — e.g. MuTect2 (or Strelka) somatic calls, VEP-annotated.
   Passed positionally to `orchestrate.py`.
2. **CNV/SV TSV (AnnotSV-merged)** — AnnotSV consumes Canvas (CNV) + Manta (SV)
   VCFs and emits a unified TSV. Passed via `--sv path/to/annotsv.tsv`.
3. **InterVar TSV** *(optional but recommended)* — pre-computed ACMG evidence
   codes for germline SNVs. Passed via `--intervar path/to/intervar.tsv`.
4. **Panel/exome BED or panel size** — for TMB denominator. Passed via `--bed`
   or `--panel-size`.

Without (2), CNV/SV findings (CDKN2A/B loss, MYC amplification, BRCA1/2 LOH,
ERBB2 amplification, BRAF fusions, etc.) cannot appear in the report regardless
of how the VCF was called.

See `inputs/README.md` for the directory conventions and file-name patterns
the regen process expects.

### Regenerating the showcase

A `make showcase-regen` target is **planned but not yet implemented** — it will
print the orchestrate command once the multi-caller inputs are staged under
`docs/showcase/inputs/`. Until then, regenerate manually:

```bash
python scripts/orchestrate.py \
    docs/showcase/inputs/codegen_777.mutect.passed.vep.vcf \
    --sv docs/showcase/inputs/codegen_777.annotsv.tsv \
    --intervar docs/showcase/inputs/codegen_777.intervar.tsv \
    --bed docs/showcase/inputs/codegen_777.panel.bed \
    --mode cancer \
    --sample-id codegen_777 \
    --clinical-board \
    --output docs/showcase/sample_codegen_777_report_v2.html \
    --json docs/showcase/sample_codegen_777_report_v2.json
```

> **Caveat — current showcase is SNV-only.** Until the combined VCF / AnnotSV
> TSV are checked in, `sample_codegen_777_report_v2.html` reflects only the
> MuTect SNV channel. CDKN2A/B loss and any other CNV/SV-only findings will be
> **invisible** in the rendered report. Reviewers comparing against external
> reports (F1, TSO500, etc.) should treat CNV/SV omissions as input-gap
> artifacts, not pipeline regressions.

## Research-reference framing

These showcase reports are **research reference documents**, not clinical
decision instruments. Every Treatment Options row cites a PMID or is marked
unverified, and narrative synthesis is generated by a local model from curated
sources (OncoKB free-tier + local CIViC). See the per-report Methodology
section for the full curation-source disclosure.
